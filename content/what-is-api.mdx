## Introduction

If you’ve ever used a mobile app, booked a cab, checked the weather, or logged into a website using Google — you’ve already used an API.

But here’s the truth:

👉 Most beginners use APIs without actually understanding them.

And that’s where confusion starts.

This guide will help you understand APIs in the simplest way possible — with real-world examples, practical explanations, and code snippets.

---

## What is an API?

API stands for **Application Programming Interface**.

In simple words:

👉 An API is a bridge that allows two software systems to communicate.

Instead of accessing a system directly, you send a request through an API and receive a response.

---

## Real-Life Analogy (Easy Way to Understand)

Think of a restaurant:

- You → Customer  
- Waiter → API  
- Kitchen → Server  

You don’t go into the kitchen.

Instead:

1. You place an order  
2. Waiter takes it to the kitchen  
3. Kitchen prepares food  
4. Waiter brings it back  

👉 The waiter is your API.

---

## How APIs Work (Step-by-Step)

Here’s what happens behind the scenes:

1. You click a button in an app  
2. A request is sent to an API  
3. API forwards request to server  
4. Server processes the request  
5. Data is fetched from database  
6. API sends response back
